# Design System — Gmail Filter Cleanup

Generated by /plan-design-review + /design-consultation on 2026-03-19.
All decisions are intentional. When in doubt, ask before deviating.

---

## Product Context

- **What this is:** A dark-mode Gmail filter cleanup utility for technical/power users
- **Who it's for:** People who want to understand and control their email setup
- **Project type:** Focused workflow tool (not a dashboard, not a marketing site)
- **Peers / reference aesthetic:** Vercel dashboard, Raycast, Linear

---

## Aesthetic Direction

- **Direction:** Industrial / Utilitarian — refined
- **Decoration level:** Minimal (typography and color do all the work)
- **Mood:** The product feels like a serious tool someone built with care — not a startup landing page, not a toy. Clarity and precision above all. The user should feel *in control*, not guided.
- **Risk:** Zero gradients, zero glow, zero glassmorphism. In 2026 when everyone uses ambient glow and blur, not having them is the differentiated choice.

---

## Color System — Dark by Default

The app is **always dark**. No light mode, no system preference detection, no toggle.

### Base Palette

| Role | Tailwind class | Hex |
|------|---------------|-----|
| Page background | `bg-gray-950` | `#030712` |
| Nav / section bg | `bg-gray-900` | `#111827` |
| Card background | `bg-gray-800` | `#1f2937` |
| Elevated / pill bg | `bg-gray-700` | `#374151` |
| Border / divider | `border-gray-700` | `#374151` |
| Border subtle | `border-gray-800` | `#1f2937` |

### Typography Colors

| Role | Tailwind class | Use |
|------|---------------|-----|
| Primary text | `text-gray-100` | Headings, body |
| Secondary text | `text-gray-400` | Subtitles, metadata |
| Tertiary / disabled | `text-gray-600` | Placeholders, footnotes, step labels (upcoming) |

**Contrast requirement:** All body text must pass WCAG AA (4.5:1 ratio). Minimum `text-gray-400` on `bg-gray-900`. Avoid `text-gray-500` on dark backgrounds — it fails AA.

## Typography

Font: **Geist** (body/UI) + **Geist Mono** (filter criteria)

Load via `next/font/google` or from the `geist` npm package (already a Next.js dependency):

```js
import { Geist, Geist_Mono } from "next/font/google"
// or: import { GeistSans } from "geist/font/sans"; import { GeistMono } from "geist/font/mono"
```

**Why Geist over Inter:** Geist has slightly eccentric character shapes (double-story 'a', distinctive numerals) that make it distinctive without being decorative. It was designed for developer tool UIs. This is a developer tool UI. Inter is ubiquitous; Geist signals a deliberate choice.

## Spacing

- **Base unit:** 4px
- **Density:** Compact (power users want more data visible)
- **Scale:** 4 / 8 / 12 / 16 / 20 / 24 / 32 / 48 / 64 / 96px
- **Tailwind:** `p-1` (4px), `p-2` (8px), `p-3` (12px), `p-4` (16px), `p-6` (24px), `p-8` (32px)

---

## Layout

- **Approach:** Single-column, centered, max 640px
- **Rationale:** This is a linear workflow tool, not a dashboard. One task, one path. No sidebar.
- **Max content width:** `max-w-2xl` (640px) for the audit flow, `max-w-md` (448px) for the landing and key-entry steps
- **Border radius:** `rounded` (4px) for pills, `rounded-md` (6px) for inputs, `rounded-lg` (8px) for cards
- **Mobile:** Identical layout — the single-column design is inherently responsive

---

## Motion

- **Approach:** Minimal-functional only
- **Step progress dots:** 100ms fill transition when a step advances (`transition-colors duration-100`)
- **State transitions** (loading → loaded, step changes): 150ms ease-out fade
- **Nothing else moves.** No entrance animations, no hover scale, no scroll effects.
- **Easing:** `ease-out` for enters, `ease-in` for exits
- **Duration reference:** micro 100ms · short 150ms · medium 250ms

## Information Architecture

### Landing Page (`/`)

Visual hierarchy:
1. Title: "Gmail Filter Cleanup" — `text-2xl font-semibold`
2. Subtitle: value prop — `text-gray-400`
3. **Before/after example**: `47 filters → 12 filters` — real anonymized data, labeled `(example from a real cleanup)` in `text-[11px] text-gray-600 italic`
4. Sign-in button (Google, **always white/light** even in dark mode)
5. Trust note: "Reads and writes Gmail filter settings only." — `text-[11px] text-gray-500`
6. **Anthropic API key note**: "AI consolidation uses Claude — you'll need an Anthropic API key. Get one free at [console.anthropic.com](https://console.anthropic.com)." — `text-[11px] text-gray-500`
7. "What it does" card

The Anthropic API key note **must be on the landing page** before sign-in, so the mid-flow key-entry step is expected rather than surprising.

### Audit Flow — Step Navigation

The nav bar shows 4 named step indicators at all times:

```
Gmail Filter Cleanup                              Sign out
─────────────────────────────────────────────────────────
  ● Audit  ● Enter Key  ○ Review  ○ Apply
  [filled = done/active, empty = upcoming]
```

- Filled dot: `bg-gray-100` (active) or `bg-blue-500` (completed)
- Empty dot: `bg-gray-600`
- Active label: `text-gray-100 font-medium`
- Upcoming label: `text-gray-600`
- The two loading states (consolidating, applying) are **not** separate nav steps — they animate within their adjacent named steps

### Step → Nav mapping

| Internal step | Nav state |
|---|---|
| `audit` | Audit = active |
| `enter-key` | Audit = done, Enter Key = active |
| `consolidating` | Audit = done, Enter Key = active (loading indicator inside Enter Key area) |
| `review` | Audit = done, Enter Key = done, Review = active |
| `applying` | Audit = done, Enter Key = done, Review = done, Apply = active (loading) |
| `done` | All = done |

## Component Specs

### Badges

Badges carry both **color AND an icon prefix** for colorblind accessibility.

| Badge | Icon | Dark classes |
|-------|------|-------------|
| Dead label | `⚠` | `bg-red-950/60 text-red-400 border border-red-800` |
| Duplicate | `=` | `bg-yellow-950/60 text-yellow-400 border border-yellow-800` |
| High confidence | none | `bg-green-950/60 text-green-400 border border-green-800` |
| Medium confidence | none | `bg-yellow-950/60 text-yellow-400 border border-yellow-800` |
| Low confidence | none | `bg-orange-950/60 text-orange-400 border border-orange-800` |

### Loading States

- Short (< 5s): spinner (`border-blue-500 border-t-transparent animate-spin`) + message
- Long (15-30s): spinner + message + italic note ("This may take 15–30 seconds…") in `text-gray-500 italic`

### Google Sign-In Button

Always use the **white/light variant** — never style it dark. The official Google button white on dark background creates intentional visual contrast that draws the eye to the CTA. Preserves Google's brand recognition.

## Copy & Tone

**Voice:** Direct, honest, a little dry. Not cheerful. Not "AI magic!" — more "here's what this does, here's what it changed."

### Key copy

| Moment | Copy |
|--------|------|
| Empty state (0 filters) | "You have no Gmail filters yet — looks like you're all set." |
| API key error (auth failure) | "[error message]\n\nGet your API key at [console.anthropic.com/keys](https://console.anthropic.com/keys)" |
| Session expired banner | "Your session expired. Please sign in again." |
| Done state headline | "Your filters have been reviewed and simplified." |
| Done state subhead | "You consolidated N rules into M — you know exactly what's in your inbox now." |
| Done state counts | "M filters created · N deleted" (centered, `text-gray-500 text-sm`) |

### Done State Design (empowerment tone)

```
     ✔  Your filters have been reviewed and simplified.

  You consolidated 12 rules into 4 —
  you know exactly what's in your inbox now.

        4 filters created  ·  12 deleted

              [ ↩ Undo all changes ]
```

The copy emphasizes: the user **reviewed** this, **understands** it, is **in control** of their filter situation. Not "the AI cleaned your mess" but "you simplified your filters, with AI assistance."

Undo button: secondary style, prominent. Not a ghost button — the user's safety net should feel solid.

---

## Accessibility

- `aria-label="Anthropic API key"` on the key input (placeholder alone is insufficient)
- Badge icons: both color AND icon communicate the meaning — never color-only
- Touch targets: minimum 44×44px on mobile
- Contrast: all body text ≥ 4.5:1 (WCAG AA)
- `lang="en"` on `<html>` — already implemented

---

## Not In Scope

| Decision | Rationale |
|----------|-----------|
| Light mode / toggle | Dark-by-default is a deliberate aesthetic choice, not a missing feature |
| `prefers-color-scheme` | ~3x implementation complexity; deferred until user demand |
| Step transition animations | Transitions are instant; no page-slide needed |
| Illustrations / iconography beyond badge icons | Not worth the asset overhead for a utility tool |
| Internationalization / RTL | English only (side project) |
| Glassmorphism, gradients, ambient glow | Explicitly excluded as AI slop anti-patterns |

---

## What Already Exists (do not reinvent)

- **`rounded-lg` radius** — consistent across all components
- **`font-mono` for filter criteria** — excellent choice, already in place
- **Diff view pattern** (before: red bg, after: green bg) — in `ProposalCard`, migrate to dark tokens
- **Badge component** — in `audit-client.tsx`, add icon prefix and dark color tokens
- **Loading spinner** — implemented, update spinner border to `border-blue-500`
- **Step state machine** — `audit | enter-key | consolidating | review | applying | done` — add nav mapping
